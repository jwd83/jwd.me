# Onyx V1 Outline And Slices

## Product Shape

Onyx is a companion publisher for an Obsidian-style Markdown folder. The primary distribution is a literal single `onyx.go` file with no third-party Go modules and no browser dependencies. A site repo can be cloned, its `docs/` folder replaced, and rebuilt with `go run onyx.go`.

Default project layout:

```txt
onyx.go
onyx.ini
index.html
docs/
public/
theme/
```

`onyx.ini` is mandatory and intentionally tiny:

```ini
site_title = My Notes
source = docs
base_url = /
```

## V1 Decisions

- Static site generation only.
- `onyx.ini` anchors the site root; Onyx searches upward from the launch directory or passed path.
- Root `index.html` is the rendered homepage.
- Homepage source priority is `docs/index.md`, then `docs/home.md`, then an autogenerated index.
- Generated pages live under `public/<vault path>/index.html`.
- Vault paths are preserved exactly on disk and URL-escaped in links.
- `public/` is rebuilt only when marked with `.onyx-generated`.
- Root `index.html` is overwritten only when missing, blank, or marked as Onyx-generated.
- Markdown notes publish by default.
- Notes with `publish: false` or `draft: true` are excluded from generated pages, nav, backlinks, and search.
- Hidden/system paths such as `.obsidian/`, `.git/`, `.trash/`, and dotfiles are ignored.
- Raw Markdown remains in `docs/`; generated pages can link back to it.
- JavaScript is progressive enhancement only.

## V1 Markdown Slice

- Frontmatter with simple `key: value` fields.
- Headings with generated anchors.
- Paragraphs, emphasis, inline code, fenced code blocks.
- Markdown links and images.
- Wikilinks with aliases, such as `[[Note|Alias]]`.
- Obsidian image/file embeds, such as `![[image.png]]`.
- Lists, task lists, blockquotes, horizontal rules.
- Minimal Obsidian callouts, such as `> [!NOTE]`.
- Tables.
- Tags are extracted for search.
- Unsupported syntax is preserved as visible escaped text rather than dropped.

## Generated Site Slice

- Sidebar navigation generated from published notes.
- Backlinks generated by inverting resolved wikilinks.
- Search generated as `public/search-index.json` plus small handwritten `public/onyx.js`.
- Default CSS and JS are embedded in `onyx.go`.
- Optional local theme overrides:
  - `theme/page.html`
  - `theme/home.html`
  - `theme/style.css`
  - `theme/onyx.js`
  - `theme/static/`, copied to `public/theme/`

## Later Slices

- Graph view using the already-built link graph.
- Better CommonMark edge-case coverage if real vaults demand it.
- Disambiguation pages for genuinely ambiguous duplicate basenames.
- More complete note embeds.
- Optional sitemap/feed generation.
